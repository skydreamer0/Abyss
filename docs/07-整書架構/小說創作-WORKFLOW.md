# 小說創作 WORKFLOW（必用流程）

> 用途：把目前專案裡已存在的規定，整理成一條可重複執行的寫作生產線。
> 適用：長篇連載、卷級規劃、S1V1 起稿。

## 0) 流程總覽（先看這張）
1. **對齊規則**：確認題材、風格三支柱、不可違反法則。
2. **鎖定輸入**：本次只處理一個層級（世界 / 卷 / 章）。
3. **產出骨架**：先做 Goal-Obstacle-Cost-Reveal-Shift-Hook。
4. **一致性檢查**：對照既有系統檔，不可偷改核心規則。
5. **深淵描寫強化**：補空間、物理異常、感官、遺骸、生物。
6. **代價落地**：每章至少一個可見代價。
7. **節奏驗收**：每 2 章小反轉、每 4 章硬代價。
8. **回寫索引**：新增設定回填對應章節與 README 索引。

## 2) 三層生產流程（Macro → Meso → Micro）

## A. Macro（卷級）
> 目標：確定一卷為何存在。

輸出格式：
- 卷目標（Volume Goal）
- 卷衝突（Core Conflict）
- 卷硬代價（Irreversible Cost）
- 卷末鉤子（Next Hook）

通過條件：
- [ ] 爽點與刀點同時存在。
- [ ] 卷末有不可逆損失。
- [ ] 能銜接下一卷。

## B. Meso（Phase 級）
> 目標：把一卷拆成四段：入局、擴壓、崩裂、收束。

每段輸出：
- 段目標
- 主要阻礙
- 關鍵決策
- 代價落點

通過條件：
- [ ] 每段至少一個「主角主動決策」。
- [ ] 至少一段出現規則反噬（非單純打鬥）。

## C. Micro（章級）
> 目標：每章都能推動讀者往下翻。

每章必填六欄（G-O-C-R-S-H）：
- **Goal**：主角本章要達成什麼。
- **Obstacle**：阻礙是什麼（人/規則/資源）。
- **Cost**：若硬闖會失去什麼。
- **Reveal**：新資訊是什麼。
- **Shift**：局勢如何翻轉。
- **Hook**：下一章追看的理由。

---

## 3) 深淵資訊「慢帶入」規則（防資訊過載）
- 每 2–3 章只推進一層架構：
  1. 現象層
  2. 流程層
  3. 勢力層
  4. 系統層
  5. 真相層
- **禁止** 同章硬塞三層以上設定。
- 新概念必須「當章可觀察」：讀者要看得到後果，不只聽角色解釋。

---

## 4) 代價與一致性檢核（寫完就跑）

### 4.1 代價檢核
- [ ] 本章是否有代價？
- [ ] 代價是否不可被下一章無痛修復？
- [ ] 代價是否影響關係/資源/記憶至少一項？

### 4.2 一致性檢核
- [ ] 是否違反既有世界規則？
- [ ] 是否新增「方便型設定」但未附代價？
- [ ] 是否和 `03-系統`、`06-生態與生物` 既有設定衝突？

### 4.3 節奏檢核
- [ ] 每 2 章至少一次小反轉。
- [ ] 每 4 章至少一次硬代價。
- [ ] 是否存在至少一個可視化名場面。

## 5) S1V1 專用 workflow（你現在最需要）

### Week 1（Chapter 1–4）
- 重點：讓讀者看懂危險與勢力進場。
- 驗收：讀者知道「深淵會污染資訊與記憶」。

### Week 2（Chapter 5–8）
- 重點：讓讀者看懂流程與技術代價。
- 驗收：讀者知道「下潛-回收-檢疫-熔斷」不是口號。

### Week 3（Chapter 9–12）
- 重點：基地級危機 + 主角禁術越線。
- 驗收：主角主動決策帶來可見代價。

### Week 4（Chapter 13–16）
- 重點：事件收束但留下不可逆損失。
- 驗收：卷末同時成立「爽點 + 刀點 + 下一卷鉤子」。
