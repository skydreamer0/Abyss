# 全書創作與正文啟動 SOP

> 目的：整合《深淵邊境》的寫作總結、正文啟動流程與發布管理規則。

---

## 1. 創作總覽與 Macro / Meso / Micro 流程

### 1.1 Pre-Flight（開工前檢查）
- **必讀**：確認風格三支柱與自我邏輯規則。
- **門檻**：確定處理層級（世界/卷/章）、主角當前目標/代價、本章揭露資訊層級（現象/流程/勢力/系統/真相）。

### 1.2 三層生產流程
- **A. Macro（卷級）**：卷目標、卷衝突、卷硬代價、卷末鉤子。
- **B. Meso（Phase 級）**：每卷拆分為入局、擴壓、崩裂、收束四段。每段需有主角決策與代價落點。
- **C. Micro（章級）**：每章必填六欄 G-O-C-R-S-H (Goal, Obstacle, Cost, Reveal, Shift, Hook)。

### 1.3 結構與資訊節奏檢核
- 核心五條：因果推進、壓力升級、Try-Fail、世界回應、能力代價。
- 分層揭露：現象層 → 流程層 → 勢力層 → 系統層 → 真相層。禁止同章硬塞超過三層設定。

---

## 2. 正文啟動標準 (DoR & DoD)

### 2.1 開稿前門檻 (Definition of Ready)
一章進入正文前，以下必須全為 Yes：
- [ ] 已填完 G-O-C-R-S-H。
- [ ] 有「因果一句話」：因為___，所以主角必須___，否則會___。
- [ ] 已標註本章壓力等級且有所升級。
- [ ] 已定義本章代價（資源/關係/記憶）。
- [ ] 已指定本章回收點與資訊來源。
- [ ] 已確認並連結本章將動用的 `settings/` 底層設定。

### 2.2 單章執行與驗收 (Definition of Done)
- **A. 骨架確認 (15m)**：拆分 3-5 場景，每場景至少 1 功能。
- **B. 正文快寫 (45m)**：先寫事件與決策，強制加入 1 次 Try-Fail。
- **C. 結構驗收 (20m)**：檢查世界回應與能力代價。
- **D. 章末交付 (10m)**：輸出摘要、下章鉤子，更新追蹤表。

### 2.3 失敗回滾機制
卡關時的處理順序：刪除無功能場景 → 補代價（失敗與損失） → 補世界回應 → 降資訊密度 → 重寫章尾。

---

## 3. 正文管理規則 (Manuscript)

正文目錄僅存放「正式文字內容」，採**一章一檔**形式管理，與所有計畫文件完全隔離。
合併至發布目錄的文件應視為「受控版本」或「最終草稿」。
- 若新增系統/生態規則，需回寫 `settings/` 目錄。
- 完成後需更新對應的 README 索引。
