# 分支與 PR 流程（合併後下一輪）

> 目的：避免「上一個 PR 已合併，但下一輪還在舊分支」造成歷史混亂。

## 1) 何時要開新分支
- 上一個 PR 已 merge。
- 你要開始新一輪內容（例如下一批章節、回寫設定、修文）。

## 2) 標準流程
1. 切回主線並更新（例如 `main`）。
2. 從最新主線開新分支。
3. 在新分支做本輪修改。
4. 提交 commit。
5. 開新 PR（不要沿用已 merge 的舊 PR 分支）。

## 3) 命名建議
- `feature/s1v1-ch17-ch20`
- `chore/workflow-sync`
- `fix/setting-consistency`

## 4) 檢查清單（開 PR 前）
- [ ] `git status` 乾淨或僅有本輪修改。
- [ ] 變更只包含本輪主題（不要夾帶上一輪殘留）。
- [ ] README/索引有同步更新（若新增文件）。
- [ ] PR 說明有：動機、變更、驗證命令。

## 5) 常見錯誤
- 在已 merge 的分支繼續開發，導致 PR 帶入舊 commit。
- 多個主題混在同一分支，審查與回滾困難。
- 新增文件未更新索引，後續找不到入口。
